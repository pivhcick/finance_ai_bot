# Быстрый старт для тестирования бота

## Шаг 1: Создайте бота в Telegram

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Придумайте имя для бота (например, "My Trading Bot")
4. Придумайте username (например, "my_trading_signals_bot")
5. Скопируйте токен, который даст вам BotFather

## Шаг 2: Настройте .env файл

```bash
# Скопируйте пример
cp .env.example .env

# Откройте файл и вставьте ваш токен
nano .env
```

В файле `.env` замените:
```
TELEGRAM_BOT_TOKEN=your_bot_token_here
```

На ваш реальный токен:
```
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

## Шаг 3: Запустите бота

```bash
# Активируйте виртуальное окружение (если еще не активировано)
source venv/bin/activate

# Запустите бота
python main.py
```

## Шаг 4: Протестируйте команды

Откройте Telegram, найдите вашего бота и протестируйте:

### 1. Начало работы
```
/start
```
Должны получить приветственное сообщение с доступными командами.

### 2. Подписка
```
/subscribe
```
Должны получить сообщение об успешной подписке.

### 3. Проверка статуса
```
/status
```
Должны увидеть информацию о вашей подписке.

### 4. Повторная подписка (проверка)
```
/subscribe
```
Должны получить сообщение, что уже подписаны.

### 5. Отписка
```
/unsubscribe
```
Должны получить подтверждение отписки.

### 6. Проверка статуса после отписки
```
/status
```
Должны увидеть, что подписка не активна.

### 7. Справка
```
/help
```
Должны получить справочную информацию.

## Проверка базы данных

После тестирования вы можете проверить, что данные сохранились в БД:

```bash
# Установите sqlite3 (если еще не установлен)
# brew install sqlite3  # для Mac
# apt-get install sqlite3  # для Linux

# Откройте базу данных
sqlite3 bot_database.db

# Посмотрите таблицы
.tables

# Посмотрите пользователей
SELECT * FROM users;

# Выход
.quit
```

## Остановка бота

Нажмите `Ctrl+C` в терминале, где запущен бот.

Вы должны увидеть:
```
INFO - Shutting down gracefully...
```

## Устранение проблем

### Ошибка: "TELEGRAM_BOT_TOKEN не установлен"
- Проверьте, что файл `.env` существует
- Проверьте, что токен указан правильно
- Перезапустите бота

### Ошибка при работе с БД
- Убедитесь, что у вас есть права на запись в текущую директорию
- Проверьте, что SQLite установлен

### Бот не отвечает в Telegram
- Проверьте, что бот запущен (нет ошибок в консоли)
- Проверьте правильность токена
- Попробуйте отправить `/start` еще раз
